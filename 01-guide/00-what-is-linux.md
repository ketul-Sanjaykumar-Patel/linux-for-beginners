# Chapter 0 — What is Linux & How It Works

## What is Linux?

Linux is an operating system kernel — the core brain between your hardware and your software. When people say "Linux," they usually mean a full Linux distribution (distro) like Ubuntu, Debian, Arch, or Fedora.
```
Your App (browser, game, code)
        ↓
   Shell / Terminal
        ↓
   Linux Kernel  ← the real Linux
        ↓
Hardware (CPU, RAM, Disk, GPU)
```

---

## Key Concepts Before We Start

| Concept | Plain English |
|---------|--------------|
| Kernel | The core of Linux. Talks to hardware directly |
| Shell | The program that reads your commands (bash, zsh) |
| Terminal | The window you type commands into |
| Distro | Linux + package manager + tools bundled together |
| Root | The superuser (admin). Has full power |
| Daemon | A background service (like a Windows "service") |
| Mount | Connecting a disk/device so Linux can use it |
| Package | A software bundle ready to install |

---

## Popular Linux Distributions

| Distro | Best For |
|--------|---------|
| Ubuntu | Beginners, huge community |
| Debian | Super stable, used on servers |
| Fedora | Cutting edge, Red Hat based |
| Arch | Manual, you build everything yourself |
| Kali | Security & hacking focused |
| Alpine | Tiny, used in Docker containers |
| RHEL/CentOS | Enterprise standard |

---

## Why Learn Linux?

- Powers **90%+ of the world's servers**
- Used in Android, smart TVs, cars, satellites
- Essential for DevOps, cloud, cybersecurity, embedded systems
- Free and open source
- Much more powerful than Windows for developers
